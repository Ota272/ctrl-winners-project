# ctrl-winners-project
# Проект: CapFinance (Telegram Mini App)
# Тип: Образовательный игровой симулятор. Платформа: Telegram Web App

# 1. Концепция и Цели
 Разработка интерактивного тренажера финансовой грамотности, интегрированного в экосистему Telegram. Проект направлен на обучение пользователей принципам управления капиталом, диверсификации рисков и реакции на рыночные изменения в игровой форме.

# 2. Технический Стек Architecture
Frontend: HTML, CSS, JavaScript

Backend: Python 

Database: SQL (SQLite, PostgreSQL).

Integration: Telegram Bot API for notifications and entry point.

# 3. Функциональные требования MVP
[] 3.1. Модуль аутентификации и профиля
Автоматическая авторизация через initData (Telegram ID).

Создание профиля пользователя с начальным балансом 

Отображение сводной статистики: общий капитал, свободные средства , PnL (Profit and Loss) за сессию.

[] 3.2. Торговый движок (Trading Core)
Реализация покупки и продажи активов различных классов риска:

Консервативные активы: Банковские депозиты , Облигации.

Рыночные активы: Акции компаний, Драгоценные металлы.

Высокорисковые/Цифровые активы: Криптовалюты, Игровые предметы (CS2 skins), Telegram Gifts.

Техническая реализация: CRUD-операции с транзакциями в SQL. Проверка достаточности средств перед каждой операцией (ACID compliance).

[] 3.3. Симуляция рынка (Market Engine)
Алгоритмическое изменение цен активов в реальном времени.

Система событий (Event System): Генерация новостных триггеров, влияющих на котировки.

[] 3.4. Геймификация и Социальные механики
Таблица лидеров (Leaderboard): Ранжирование пользователей по общему размеру капитала. Реализация через SQL-запросы с сортировкой и лимитом

Случайные события (Risk Mechanics): Механика «Рулетка/Риск», позволяющая мгновенно выиграть или потерять часть капитала (демонстрация рисков азартных игр в инвестировании).

# 4. Структура Базы Данных (ER-Model Outline)
Для реализации MVP используется реляционная модель данных:

users: Хранение данных пользователя (tg_id, username, balance_fiat, registration_date).

assets: Справочник активов (id, name, type, current_price, volatility_index, image_url).

user_portfolio: Связующая таблица владения (user_id, asset_id, quantity, average_buy_price).

transactions: Лог операций для аналитики и истории (id, user_id, type, amount, timestamp).
